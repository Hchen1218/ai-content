# Skill: 生成周运营复盘

## 触发条件
用户说以下内容时触发：
- 「生成周运营复盘」
- 「这周复盘一下」
- 「给我本周运营复盘」

## 执行步骤
1. 读取 `01-内容生产/数据统计/内容数据表.md`
2. 读取 `02-业务运营/业务规划/📋 进行中的运营动作.md`
3. 读取 `02-业务运营/业务规划/✅ 已完成的运营动作.md`
4. 在 `02-业务运营/业务规划/周期复盘/` 新建当周复盘文件
5. 复盘至少包含：
   - 本周结果
   - 关键问题
   - 进行中动作判断
   - 下周建议动作
   - 停做建议
   - 本周结论

## DBS 辅助深诊
只有在以下解释层问题上，允许补充调用 `/dbskill`：
- 为什么卡住
- 为什么同题不同平台失效
- 为什么动作推进不了
- 为什么明明有方向，但执行持续拖延

统一写法：
- `先调用 /dbskill 做深度分析`

这一步只补解释，不替代复盘事实源。

## 注意事项
- 复盘不是情绪总结，而是运营判断
- 结论要尽量落到平台、题材、形式三个维度
- 如果数据不足，明确写“样本不足，继续验证”
- 数据和已完成动作仍然是唯一事实源
- DBS 结论不能覆盖指标阈值、数据事实和动作结果
