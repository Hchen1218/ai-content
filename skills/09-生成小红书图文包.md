# Skill: 生成小红书图文包

## 触发条件
用户说以下内容时触发：
- 「生成小红书图文包」
- 「把这条内容改成小红书图文」
- 「做一篇小红书图文笔记」
- 「这条要不要转图文」
- 「按小红书图文模板做」

## 第一原则
- 不预设模板族，先判断内容是否值得做成图文
- 不照抄对标内容，只复用格式和转化逻辑
- 输出必须是 `可发包`，不能只给结构骨架

## 澄清规则
如果用户的需求仍然模糊，必须先做澄清。

澄清时只允许：
- **一次只问一个问题**
- 根据回答继续追问
- 直到你对这条图文的目标、受众、转化目标有 **95% 把握**
- 在达到这个把握前，不给最终方案

优先澄清顺序：
1. 这条图文最终想要什么结果：点击 / 收藏 / 评论 / 搜索
2. 面向谁：小白 / 已入门用户 / AI 从业者
3. 这条内容更像：教程 / 科普 / 观点
4. 有无现成源内容：视频稿 / 已发布内容 / 素材库笔记

## 执行步骤

### 1. 判断是否值得转图文
先读取源内容和历史数据，判断：
- 这条内容是否适合改成小红书图文
- 它更适合什么样的页卡节奏和信息密度
- 对标里应该借哪一部分：封面、页卡节奏、截图占比、文字留白

### 2. 读取相关方法论
读取：
- 如涉及标题，再读 `01-内容生产/方法论/小红书标题方法论.md`
- 如用户给了对标，再读对应拆解或截图

### 3. 生成图文可发包
输出必须包含：
- 选中的源内容
- 是否建议转图文
- 判断依据
- 主标题 1 个，备选标题 3 个
- 封面文案 + Canva 排版动作
- 逐页页卡表：
  - 页号
  - 页面目标
  - 上屏文字
  - 设计元素
  - 参考对标页型
  - 素材来源建议
  - 如果需要 Gemini 出图，直接给 prompt
- 正文
- tag
- 置顶评论
- 发布前检查清单

### 4. 图片与排版规则
- 默认排版工具：`Canva`
- 工具界面、官网、新闻、产品功能：优先真实截图 / 网图
- 概念解释图、类比图、抽象示意图：给 Gemini prompt
- 不要输出“和对标一样的内容图”
- 只输出“同样设计作用的图”
- 不让 AI 生成大段中文排版文字，文字后期手动排

## 输出标准

### 必须做到
- 每一页都说清楚“这一页干什么”
- 每一页都说清楚“这一页用什么图”
- 每一页都说清楚“在 Canva 里怎么排”
- 所有 prompt 都可直接复制到 Gemini

### 不允许
- 只给页型，不给上屏文字
- 只给文案，不给配图方式
- 只给思路，不给封面文案和正文
- 按对标原内容做改写

## 注意事项
- 如果判断“不建议转图文”，先把这个结论明确告诉用户，再说明原因
- 如果内容已经是图文，不要再按“视频转图文”处理
- 你可以借鉴对标的设计元素，但不能把对标内容本身换词复写
