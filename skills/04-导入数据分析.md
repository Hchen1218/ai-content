# Skill: 导入数据分析

## 触发条件
用户说以下内容时触发：
- 「导入数据」
- 「分析数据」
- 「数据复盘」
- 「我导出了新数据」
- 「帮我看看数据」

## 输入要求
- 小红书或抖音创作者平台导出的 xlsx 文件
- 默认位置：用户桌面
- 用户需说明是哪个平台的数据

## 执行步骤

### Step 0：导入前口径检查（小红书必做）
先运行口径守卫脚本，确保账号口径和单条口径被明确区分：

```bash
python3 "03-工具与效率/常用工具/xhs_metric_guard.py" "/path/to/近30日观看数据.xlsx" \
  --md "01-内容生产/数据统计/内容数据表.md" \
  --strict
```

若命令返回非 0，说明有风险项（如缺少关键标签、单条点击率缺失、未识别账号封面点击率），先修正后再导入。

### Step 1：读取数据
1. 用 Python（openpyxl）读取 xlsx 文件
2. 识别所有 sheet 名称和数据结构
3. 提取账号概览、每日趋势、笔记列表

### Step 2：更新内容数据表
目标文件：`01-内容生产/数据统计/内容数据表.md`

1. **账号概览**：更新为最新数据，并明确拆分点击相关指标：
   - `平台封面点击率（账号口径）`
   - `观看/曝光比（自算口径）`
   - `内容点击率（单条口径）`
2. **每日趋势**：替换为新的 7 天数据
3. **单条内容明细**：
   - 新增内容录入到视频或图文表格
   - 已有内容更新数据
   - 单条表头统一写为 `内容点击率（单条口径）`
4. **平均数据参考**：重新计算各体裁平均值
5. **跨平台对比**：如有同内容在多平台发布，更新对比表

### Step 3：更新已发布内容归档
1. 匹配 `01-内容生产/选题管理/03-已发布选题/` 对应文件
2. 更新顶部「数据表现」区块
3. 若有实际发布文案或封面标题缺失，按用户要求补齐

### Step 4：输出运营层结果
目标目录：`02-业务运营/业务规划/`

1. 读取 `📋 进行中的运营动作.md`
2. 优先按 `Content Link` 匹配内容，没有再按标题关键词弱匹配
3. 为命中的动作生成：
   - `Actual Result` 草稿
   - `建议 Decision`
   - `建议状态`
4. 在 `周期复盘/` 生成当周周运营复盘
5. 不要自动把动作移到 `✅ 已完成的运营动作.md`，只有用户明确说「完成运营动作」时才归档

### Step 5：识别爆款
判断标准：**观看量 > 当前平均值的 2 倍**

如果发现爆款：
1. 提示用户是否需要归档到 `素材库/爆款文稿库/`
2. 需要用户提供完整文稿（口播文案 / 图文正文）
3. 按既有格式归档
4. 分析爆款原因
5. 提取可复用结构

### Step 6：成长趋势分析
对比历史快照数据：
1. 核心指标环比变化（观看、曝光、平台封面点击率）
2. 内容策略是否有效
3. 哪些类型的内容在持续增长
4. 哪些运营动作正在被验证，哪些需要停止

### Step 7：选题与动作建议（数据充足时）
当积累了 **3 次以上快照数据** 后，同时给出：
1. **内容建议**：哪些题材值得继续做
2. **动作建议**：哪些运营动作值得创建或继续
3. **平台建议**：哪些题材应该偏向抖音，哪些应偏向小红书

## 输出格式

```text
📊 数据导入完成！

### 本期概览
- 平台：[小红书/抖音]
- 数据周期：[开始日期] ~ [结束日期]
- 新增内容：[N] 条

### 核心指标变化
| 指标 | 上期 | 本期 | 变化 |
|------|------|------|------|
| ... | ... | ... | ... |

### 亮点
- [爆款/高互动内容概述]

### 运营动作更新
- [命中的动作 + 建议 Decision]

### 发现
- [标题规律 / 平台适配 / 题材判断]

### 建议
- [下一步运营动作建议]
```

## 注意事项
- 先读取现有数据表，了解历史数据结构
- 不要无脑覆盖已有内容资产，尤其是口播稿、封面标题、正文
- 小红书点击指标必须区分口径并明确标注
- 小红书导入前必须先运行 `xhs_metric_guard.py --strict`
- 爆款归档需要用户确认并提供完整文稿
- 周复盘是半自动生成，不等于动作已归档
