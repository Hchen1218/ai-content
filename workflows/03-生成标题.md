# Skill: 生成标题

## 触发条件
用户说以下内容时触发：
- 「生成标题」
- 「小红书标题」
- 「帮我想个标题」
- 「标题怎么写」

## 执行步骤
1. 先判断内容方向、内容形式、核心表达是否已经锁定
2. 如果用户只给了模糊选题，或标题问题本质上还是内容问题，**先调用 `/dbskill` 做深度分析**
3. 在内容方向已经锁定的前提下，理解内容主题和核心卖点
4. 读取 `方法论/小红书标题方法论.md`
5. 应用多个标题公式生成选项
6. 为每个标题解释逻辑
7. 推荐最优选项

## DBS 介入条件
以下情况不要直接生成标题，先 `/dbskill`：
- 用户只给了一个大方向，没有明确内容主张
- 标题写不出来，其实是因为概念没讲清或角度没定下来
- 用户已有短视频稿，但前 5 秒抓力弱，标题问题本质上是内容诊断问题
- 用户在纠结“到底做图文还是视频”“这条该讲什么重点”

## 调用边界
- `/dbskill` 负责深度分析，不负责替代标题产出
- 一旦内容方向锁定，当前 Skill 仍按本地标题方法论输出 3 个标题选项

## 输出格式
```
## 标题选项

### 选项 1（推荐）
[标题]
- 公式：[使用的公式]
- 逻辑：[为什么这样写]

### 选项 2
[标题]
- 公式：[使用的公式]
- 逻辑：[为什么这样写]

### 选项 3
[标题]
- 公式：[使用的公式]
- 逻辑：[为什么这样写]

## 推荐理由
[为什么推荐选项1]
```

## 示例对话

**用户**：帮我生成标题，内容是关于用 Claude 写小红书文案

**AI**：
## 标题选项

### 选项 1（推荐）
**3步教你用Claude写出爆款小红书文案**
- 公式：数字+结果
- 逻辑：数字降低阅读门槛，"爆款"是用户想要的结果

### 选项 2
**还在为写文案发愁？Claude 帮你10分钟搞定**
- 公式：痛点+解决方案
- 逻辑：直击用户痛点，给出明确解决方案

### 选项 3
**我用 Claude 写文案，粉丝问我请了什么写手**
- 公式：故事/结果导向
- 逻辑：用故事引发好奇，暗示效果很好

## 推荐理由
选项1最直接，数字型标题在小红书点击率通常更高，同时"爆款"这个词符合用户追求。

## 注意事项
- 小红书标题控制在 20 字以内
- 优先使用数据统计中验证过的高点击率关键词
- 避免使用数据统计中记录的低点击率陷阱
- 如果标题问题本质上是内容问题，先 `/dbskill`，不要硬生成
