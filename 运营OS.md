# 运营OS

## 项目定位

这是 `ai-content` 的总操作系统入口。

它不是单纯的内容生产说明，而是把你的账号拆成两层系统来运行：
- `01-内容生产/`：负责选题、素材、标题、发布与已发布归档
- `02-业务运营/业务规划/`：负责运营动作、周复盘、账号能力画像、平台策略更新

**目标**：把“想到什么就做什么”升级为“导入数据后，系统自动给出下一步运营动作”。

## 双系统分工

### 内容系统负责什么
- 记录选题
- 深化选题
- 检索素材
- 生成标题
- 发布后归档到 `03-已发布选题/`
- 更新 `数据统计/内容数据表.md`

### 运营系统负责什么
- 把本周要验证的增长动作写成卡片
- 给每个动作设置 `How to measure`
- 数据导入后匹配动作，生成结果草稿
- 形成周复盘，反推哪些动作要放大、继续、停止
- 定期更新平台策略和账号能力画像

## 核心工作流

### 1. 记录选题
当用户说「记录选题」或「记录一个想法」时：
- 将想法追加到 `01-内容生产/选题管理/00-选题记录.md`
- 自动添加当前日期
- 来源标记为用户指定或「灵感」
- 潜力默认为 ⭐⭐⭐

### 2. 深化选题
当用户说「深化选题 [选题名]」时：

#### 第零步：需求澄清（必须先做，不可跳过）
在做任何事情之前，先用追问法搞清楚用户的真实意图：
- 目标平台（抖音 / 小红书 / 即刻）
- 内容形式（口播视频 / 图文 / 白板笔记）
- 目标用户及核心痛点
- 差异化角度
- 是否已有参考来源

#### 第一步：检索素材（澄清后执行）
- 必须先执行「检索素材」
- 在 `01-内容生产/选题管理/01-待深化选题/` 创建选题卡片
- 使用项目模板填写基本信息和素材链接

### 3. 检索素材
当用户说「检索素材」或在创作任何内容之前：
- 先搜索 `素材库/AI工具库/`
- 再搜索 `素材库/观点库/`
- 再搜索 `素材库/金句库/`
- 最后搜索 `素材库/爆款文稿库/`
- 找到相关内容时优先复用，不要重造轮子

### 4. 生成标题
当用户说「生成标题」或「小红书标题」时：
- 参考 `01-内容生产/方法论/小红书标题方法论.md`
- 生成 3 个标题选项
- 解释每个标题的逻辑

### 4.1 生成小红书图文包
当用户说「生成小红书图文包」或「把这条改成小红书图文」时：
- 必须先读取 `01-内容生产/方法论/小红书图文转化方法论.md`
- 先判断这条内容是否适合转图文
- 再匹配正确的图文模版
- 再盘点现有素材是否足够支撑页卡
- 若素材不足，先补：
  - 网上观点
  - 案例
  - 截图
  - 数据或评论区问题
- 最后再输出 `Canva 可执行图文包`
- 不允许把视频逐字稿直接切成图文
- 输出必须包含：
  - 转化判断
  - 模版匹配
  - 素材盘点
  - 补料建议
  - 页卡结构
  - 完整图文成稿

### 5. 记录数据 / 归档已发布内容
当用户给出抖音或小红书导出的表格，或说「记录数据」时：

#### 第一步：匹配选题
- 遍历 `01-内容生产/选题管理/01-待深化选题/`
- 用内容相似度把表格中的内容匹配到已有选题
- 匹配不到的内容，直接在 `03-已发布选题/` 新建归档文件

#### 第二步：归档到已发布
- 将匹配到的文件从 `01-待深化选题/` 移动到 `03-已发布选题/`
- 文件名改为实际发布日期，如 `20260306-API科普-一根水管接三台电器.md`
- 保留策划内容，替换成实际发布文案，补写数据表现区块

#### 第三步：更新数据统计
- 同步更新 `01-内容生产/数据统计/内容数据表.md`
- 添加或更新对应平台的单条内容明细
- 如有账号概览数据，新增历史快照
- 小红书导入前必须先运行：
  - `python3 "03-工具与效率/常用工具/xhs_metric_guard.py" "<小红书xlsx路径>" --md "01-内容生产/数据统计/内容数据表.md" --strict`
  - 若返回非 0，先修正风险项再写入主表
- 小红书点击率必须拆成：
  - `平台封面点击率（账号口径）`
  - `观看/曝光比（自算口径）`
  - `内容点击率（单条口径）`

#### 第四步：输出运营层结果
- 读取 `02-业务运营/业务规划/📋 进行中的运营动作.md`
- 先按 `Content Link` 匹配内容，再按标题关键词弱匹配
- 为命中的动作生成：
  - `Actual Result` 草稿
  - `建议 Decision`
  - `建议状态`
- 在 `02-业务运营/业务规划/周期复盘/` 生成当周周复盘

#### 第五步：爆款归档（如适用）
- 如果数据超过平均值 2 倍，同步归档到 `素材库/爆款文稿库/`
- 分析爆款原因，提取可复用结构

## 运营规则

### 1. 内容不是目标，动作才是目标
每条内容都应该服务于某个运营动作：
- 拉新
- 涨粉
- 收藏
- 搜索
- 人设
- 验证

### 2. 没有 `How to measure` 的动作不进入执行
运营动作卡必须写清楚：
- 想验证什么
- 多少数据算达标
- 哪种结果说明应该停止

### 3. 已完成动作是唯一的策略事实源
平台策略、账号能力、方法论更新，都只能从：
- `✅ 已完成的运营动作.md`
- `01-内容生产/数据统计/内容数据表.md`
反推，不靠主观印象。

### 4. 小红书与抖音不追求同质化
当前默认平台分工：
- 抖音：拉新 + 放大
- 小红书：搜索承接 + 收藏沉淀

## 快速指令

| 说 | 做 |
|---|---|
| 记录选题 | 添加想法到选题记录表 |
| 深化选题 | 先澄清需求 → 检索素材 → 创建选题卡片 |
| 检索素材 | 搜索素材库找可复用内容 |
| 生成标题 | 基于方法论生成 3 个标题 |
| 判断图文转化 | 判断是否适合转图文 -> 匹配模版 -> 盘点素材 -> 补料建议 |
| 生成小红书图文包 | 判断是否适合转图文 -> 输出 Canva 可执行图文包 |
| 记录数据 | 匹配选题 → 归档到已发布 → 更新数据表 → 输出运营复盘草稿 |
| 创建运营动作 | 在 `📋 进行中的运营动作.md` 新建本周动作 |
| 完成运营动作 | 把动作移动到 `✅ 已完成的运营动作.md` |
| 生成周运营复盘 | 输出本周结果、问题、下周动作 |
| 更新账号策略 | 基于已完成动作和数据更新策略与能力画像 |
| 查看账号打法 | 汇总当前平台角色、强项、停做项 |
