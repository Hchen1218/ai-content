### 0.2 关键指标目标值（用你 repo 的方法论设定）
参考: `01-内容生产/数据统计/内容数据表.md`

- 收藏率: >= 6%（你的爆款/强内容区间 6%+）

### 0.3 成功判定（YES/NO）

- 后续系列选题中至少 1 条进入 2 级流量池（以曝光显著抬升为信号）

- 定义“爆款/viral”（7天口径，满足任一即可）:
  - 曝光 >= 10,000
  - 收藏 >= 300
  - 涨粉 >= 50
- 移动端验收: 视频里必须展示一次手机端浏览效果（或H5自适应），避免“手机打开很丑”的反噬。

## 1) 约束与风险（必须）

### 1.1 隐私约束
- 不能展示真实公司/项目/联系方式/站点域名等敏感信息。
- 网站内容允许占位符，但必须“看起来真实、可复制”。

### 1.2 合规与平台风险
依据（外部研究综合）:
- 文末标注“AI辅助创作”；AI图/AI生成要可解释来源。
- 避免在笔记正文直接强导流外链；优先“评论区置顶/私信关键词”。

### 1.3 内容风险
- 只展示成果不讲方法 → 收藏低。
- 只讲教程不展示成果 → 停留低、关注低。
- 情绪共鸣标题 CTR 可能高但关注转化差（你历史验证）。

### 1.4 隐私替身规则（必须，执行前先写好）
- 敏感字段清单（拍摄与网站内容都适用）:
  - 姓名/头像/手机号/邮箱/微信号/地址
  - 公司名/学校名/项目名/域名/GitHub用户名
  - 浏览器标签页/通知栏/书签栏/IDE侧边栏文件路径/终端历史/API Key
- 替换策略（固定选一种，避免每条内容不一致）:
  - 公司: “某头部互联网公司/某ToB软件公司/某出海团队”
  - 项目: “DAU百万级/从0到1/跨部门协作/交付周期X周”
  - 联系方式: 默认不直出，统一改为“联系表单/私信关键词”
  - 站点域名: 统一用 `demo.yourproduct.com`（不要出现真实域名）
  - 头像: AI生成/插画；不要用真人照片（或用局部遮挡）

### 1.5 屏录防泄露清单（必须）
- 录屏前:
  - 关闭无关应用与浏览器标签；隐藏书签栏；关通知
  - IDE：隐藏侧边栏可能暴露用户名/路径的区域；准备 demo workspace
  - 终端：清空历史或改用新窗口；确保无 token 输出
- 录屏中:
  - 出现异常弹窗/提示 → 立刻停录重来
- 录屏后:
  - 回放检查 2 遍：1) 信息泄露 2) 链接/二维码/账号id

## 3) 第 1 条视频（成果展示向）— 决策完整脚本包

### 3.1 视频规格
- 时长: 60-90s
- 画面结构: 70%屏录 + 30%口播（可不露脸，但必须有稳定配音）
- 字幕: 必须（白底黑字/黑底白字都可，确保可读）
- 节奏: 每 1-2 秒至少一个视觉变化（切镜/放大/高亮/弹字）

### 3.1.1 A/B 测试（只做标题/封面，不改正文）
目的: 你的历史数据显示“CTR高 ≠ 涨粉高”，用封面/标题做最小成本测试。
- A版（推荐）: 数字+反直觉
- B版: 零门槛承诺
执行: 同一视频只能选一个版本；另一个版本用于“次日补发图文/系列第2条”测试（避免重复内容被判同质）。

### 3.2 镜头清单（shotlist，按时间轴）
0-3s（Hook）
- 镜头: 网站成品首页 1 秒 + 关键模块快速闪回（About/Skills/Contact）
- 叠字: “0代码 | 我用AI做了个人网站”

3-10s（结果+圈定受众）
- 镜头: 你在 IDE 里对话的界面（隐藏个人信息），同时右侧预览网站
- 口播: “我是XX（项目经理/不会写代码也行），用 vibe coding 做了个人网站。”

10-55s（3个反直觉技巧，每个 12-15s）
- 技巧1: “审美不用自己想：用 2 张参考图锁定风格”
  - 演示: 参考网站截图 → AI 生成结果（before/after）
  - 注意: 引导观众“收藏截图清单”

- 技巧2: “先做信息结构，不要先做动效”
  - 演示: 首屏三行文案模板（我是谁/我帮谁做什么/一个可信证明）
  - 输出一条模板句式（可复制）

- 技巧3: “让网站从能看变惊艳：用 3 条 prompt 迭代细节（字距/留白/对齐/按钮）”
  - 演示: 选中元素 → 对话修改 → 预览变化

55-90s（CTA）
- 置顶资源: “我用的 prompts + 模板清单 + 首屏三行文案模板”

- 互动提问: “你想做哪种？简历站/作品集/个人博客，评论扣 1/2/3”

  - 模板市场
  - 自定义CSS/JS
  - 访问统计
  - 仅PDF解析路径（必须有文本兜底）
