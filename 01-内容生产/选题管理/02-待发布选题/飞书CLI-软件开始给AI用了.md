## 选题分析

### 目标用户
- 已经在用 Claude Code、Codex、OpenCode 这类 Agent 的重度 AI 用户
- 平时高频使用飞书、多维表格、飞书消息和协同功能的内容团队或小团队
- 对 Agent 时代的软件变化有兴趣，但不想听纯技术科普的普通用户

### 核心价值
- 把 `CLI` 这种容易把人劝退的术语，翻译成普通人能听懂的人话
- 让观众第一次直观理解，什么叫“AI 不只是聊天框，它开始直接操作软件”
- 用 3 个具体场景把“飞书开始给 AI 用了”这个判断立住

### 爆款公式对照
- ✅ 认知升级：飞书不是多了个工具，而是开始认真给 AI 开入口
- ✅ 人话解释：GUI 给人点，CLI 给命令和 AI 走
- ✅ 体验承接：不是新闻播报，是“我装完之后真的跑通了”
- ✅ 具体场景：发消息 / 查表 / 处理数据
- ✅ 收藏理由：结尾给安装 prompt，观众可以直接拿去试

### 这条内容不该怎么做
- 不要做成“飞书 CLI 安装教程”
- 不要用 `CLI / GUI` 纯概念硬讲 40 秒
- 不要把安装命令做成主钩子，安装 prompt 只能做轻奖励

## 完整口播文案（当前定稿）

飞书这次开源 CLI，我装完之后第一个反应挺直接的。  
以后我用飞书，可能真不用自己点那么多下了。

以前的飞书很简单。  
你自己打开，自己找人，自己翻表，自己发消息。  
所有动作都得你亲手来。

现在不一样了。  
我可以直接把一句话丢给 Claude Code，让它去飞书里办事。  
那一瞬间的感觉很奇怪，也很新。  
你会发现，AI 已经不只是坐在聊天框里陪你说话了。它开始真的动你的软件了。

如果你本来就在用 Claude Code，后面我把我自己装飞书 CLI 的那句 prompt 放出来。  
直接复制给 Agent 就行。CLI 和那 19 个 skills 会一起装上。  
第一次卡住的地方其实不是安装，是你根本不知道那句话该怎么说。

很多人看到 CLI 会下意识地觉得麻烦。  
其实真没那么远。  
你平时看到的飞书页面，是给人用的。  
CLI 像另一条入口。  
不用点来点去，直接下命令。  
AI 也能走这条路。

所以这次更新有意思的地方，不是“飞书又发了个面向开发者的工具”。  
而是它开始让 Agent 直接碰它自己的核心功能了。

我装好之后，先试了个最直观的事。  
发消息。

我让 Claude Code 去飞书通讯录里找人，再把一段话发给同事。  
它会自己排掉我自己，自己匹配联系人，再一条条发出去。  
这件事厉害的地方不只是省时间。  
而是你会突然反应过来，它不是在帮你“想怎么发”，它是在替你发。

第二个场景，是查表。

如果你平时用飞书多维表格，应该都知道那种烦。  
自动化做了一堆，最后还是会有很多字段得手填。  
有人漏了，有人填错了，有人一直拖着不填。  
以前这种事只能靠人自己盯。  
现在你可以把规则告诉 Claude Code，让它去扫表。  
哪里空着，哪里不对，谁该补，它都能先帮你找出来。

这个时候，味道就变了。  
它已经不只是个“效率工具”，而是在替你接一些原来得靠人盯着的事。

第三个场景，是处理数据。

以前飞书里的表，大多数时候就是存着。  
真要分析，还得自己导出来，再折腾一遍。  
现在它可以直接去读这些数据，再帮你整理成网页、日报，或者一个更顺手的页面。  
这一步很关键。  
因为它不是只会“查”，它开始会接着往下做了。

所以我现在对这次更新的感觉很明确。  
CLI 只是表面。  
真正变的，是软件开始给 AI 留入口了。

以前是你自己上手操作软件。  
以后很可能会越来越多地变成，你开口，AI 去替你操作。  
人还是那个提需求的人。  
但真正去翻表、发消息、找数据、跑流程的，慢慢会换成 AI。

飞书这次真正让我觉得不一样的，就是这里。  
它不像是在单纯加一个功能。  
它更像是在提前适应一件马上会发生的事。

软件不只是给人用。  
也得给 AI 用。
