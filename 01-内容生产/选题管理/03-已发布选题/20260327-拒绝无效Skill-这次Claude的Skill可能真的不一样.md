你和我一样做 skill 给我最烦的地方，并不是写不出来，而是写出来了，不知道它到底有没有用，到底怎么触发。

这次更新的 skill creator 2.0 倒不是更会写 skill 了，而是它开始帮你测 skill、改 skill、对比 skill，完成整个生产链的闭环了。

我拿自己的几个思路跑了一遍以后，我最大的感受就是：你手里的所有 skill，都值得用它重新跑一遍。

如果你还不知道 skill 是什么，你可以把它理解成是挂在你 agent 上一个技能包，或者是一个固定的工作流。1.0 其实就是总结你的工作步骤、工作流程，创建一个 VB 的文件。但是创建好以后，它的触发方式和运行方式都还只是一个黑盒状态，你需要通过不断地使用、不断地调试，来完善你的这个 skill。

那 skill creator 2.0 的特点就是，在你身上玩这个 skill 以后，它会帮你跑一遍整个流程，然后根据里面遇到的问题先去帮你改，进一版 skill；再根据你库里的 skill 或者你大模型的技能，来判断是在你的 skill 上加一个分支，或是新建一个 skill。完成这一整套流程以后，它才会创建一个 skill。

更新的方式也非常简单，你只需要把这一行文字原封不动地发给你的 Claude Code，或者去让你的 agent 帮你更新就好了。大家可以暂停先复制。后续我会拿一个真实的给你看，这个 skill creator 为什么值得更新。

接下来看一下我的这个 skill，它叫 viral content breakdown。就是我做自媒体的时候，看到一些非常棒的对标或者一些爆款内容，我会把这个链接交给我的 agent，然后让它去帮我把这个视频的内容扒下来，做一个整体分析，给我一些创作上面的灵感和指导等等。

差不多说完，我们现在开始。首先我先让这个 skill creator 评估了一下我的这个 skill。按照这个规则看，它给了一个评分，四个评分项：第一是成品度 8~10 分；第二个就是触发设计，可能就是我触发的方式不稳定，你发个链接或者说个话，不能保证它完全能读上；第三个就是鲁棒性，也就是抗干扰能力，发一些别的东西会不会误触。你可以把它理解成一个稳定性测试吧。下面的一堆问题其实看不看得懂无所谓，直接跟它说开始优化。

然后它会告诉你，它强化了、修正了很多东西。看不懂也无所谓。我是一个不懂代码的小白，我就让它用大白话告诉我到底优化了什么东西。它说优化了三件事。

第一件事，让这个 skill 更容易被用上了。在它刚开始评分的时候，它的触发性是给了分的，这一点我觉得非常不错。因为我每次都要去跟它说“帮我分析一下这个”，它才会触发，有点笨笨的。

第二个就是修复了一个误导系统的问题，就是刚刚说的假阳性。其实你根本没有登录上去，但它就说自己登录好了，后续在操作的时候它就会触发幻觉或者进入别的流程，影响整个系统的稳定性。

第三个就是增强了一些抓取的备用方案。换句话来说，就是加强了它刚刚那个 robustness。

第四，它补了一版考试题。它给这个 skill 设了几个典型的任务，方便去测试它到底是不是真的做到了一个稳定的产出。它还会给你提意见，就是还要优化的还有三块：第一，做一个复检，来看它整个完成的情况到底怎么样；第二，让我做一个真实的压力测试，拿几个真实的链接跑一下；第三，继续优化它的分析质量。

其实在我的使用过程中，它主要起到一个抓取的作用，我只是单纯看它的内容而已，它的分析内容我是不看的。但是它帮我准确提出来了：我的分析内容不够稳定、不够准。它给了我一个优化建议。然后我就让它继续优化。

看一下它优化了什么。它把考试题补成了能打分的考试题。它之前是写了五个场景测试，现在给每一个场景都添加了明确的检查点。第二，它加了一个新的脚本，做一个复检：证据有没有带上，主页结构，Top 3 的推理是否完善。第三个就是它把报告内容优化了，让它少一点套话，多一点真实内容。

反正它在短短五分钟里就已经自我迭代了两轮了。它有两组本地的模拟数据做了验证，做了一组视频、做了一组图文，然后新的分析器、正常质检脚本判定通过。接下来它建议我进入最重要的一步，就是做一个真实的链接测试，把新版旧版结果放在一起对比。

接下来我把我自己的一个链接丢给它测试一下。这个链接已经跑通了，给我成功生成了报告，结果检查通过，给了我一个最终报告。它还告诉我了一个顺手学到的东西，就是之前封面标题会被错误识别成乱码，在正式设置里面它直接给修好了；之前互动数据拿不到，报告解释不够直白，现在它会直接说平台没返回，所以我不能猜、不会编数据。

它还告诉我了一个还要优化的地方，但是我没有管了，我不想再优化了，我想看看它优化前后到底差在哪。它噼里啪啦优化了一大堆，又是 readme、又是 json、又是 markdown 的内容，你不想看就太枯燥了。那也很简单，你就跟它说：把优化前后对比，帮我做一个 HTML 的可视化报告。它就会给你做一个网页结构的报告。

接下来我们一起来看一下这个报告。计划内容是同样一个内容，在优化前和优化后一个拆解结果。这里附了四项具体优化的结果，细项样本标题这是我的一个内容，大家感兴趣的话可以去看一下。对比维度从三段变成了五段可读性的结论，给了一堆术语的识别。因为我的是视频，我原话说的是 prompt，它之前那一版识别成了 prop，后来修正成了 prompt。把一些识别错误也修正了。开场 hook 也从截断句变成了完整的一个冲击句。

以后我们看它具体的形式结构，这里有一个 hook 的对比。优化前它只是单纯把这个 hook 给扒下来了，而且还不够完整，我这句原话不是这么说的。优化后它把这个完整的 hook 识别出来了，识别率上去了。第二个就是它的叙事结构做了更详细化的展开。第三个同样也是做了一个细化，在以后的分析过程中可能会更加顺畅。脚本部分我就不展开了，大家想看可以去翻我原来的内容。然后是爆点驱动，这个相对来说给得比较简单、比较套话。它直接给出了我的主体三个技巧，还有量化结果、可信度、冲击力等等。然后是改写建议，旧版只是保留了一个形式，新版是让我保留一个格式的钩子，数字加结果。

skill 这个概念，作为近两年 AI 界最有用且最火爆的概念，它的出现无疑是具有里程碑式意义的。这次发布的 skill creator 2.0，并不是改变了做 skill 的入口或者写 skill 的方式，而是改变了整个 skill 生产的过程。从做出来是黑盒的状态，变成了有生产、测试、优化、比较这一整个流程的闭环，它能极大提高你生产和使用 skill 的效率。

同时，这个 skill creator 也不只是针对 create 这个板块，也针对 improve 这个板块。它可以针对你所有的老 skill 给你做一轮新的评估、新的优化。这也是我为什么会说，你现在手里所有的 skill 都值得用它重新再跑一遍。
