## 完整口播文案（补录）

我装完飞书刚刚开源的 CLI 后，我深刻地感觉到，软件这回真的不是给人用的了。

以前用飞书，就是你自己上去点。发消息、翻表格、找联系人，都是你自己来。
装完以后，可以直接把一句话丢给 Claude Code，让它去飞书里干活。

如果你本来就在用 Claude Code 或者 Codex 这种 Agent，我会在评论区把我自己装飞书 CLI 的那句 prompt 放出来。直接复制给 Agent 就行，整个 CLI 连同配套的 19 个 skills 会一起装好。

稍微科普一下，免得看到 CLI 这三个字会觉得很劝退。其实很简单，你平时看到的飞书页面，那些图标布局是给人用的，叫 GUI；而 CLI 更像另一条通道，是命令行形式的，AI 也能用。

所以这次更新，并不是飞书更新了什么功能，而是它开始给 AI 留入口了。

我装好之后，先试了一个最直观的场景，发消息。
我直接让 Claude 去通讯录里找人，然后给同事发一段话。它会自己排掉我自己，再去匹配联系人，然后一条条发出去。不知道你们能不能理解，但是这种感觉真的很奇妙。

第二个场景，是查表。
如果你平时用多维表格、云文档，应该都知道，经常会有人漏填，有人填错了，或者一直拖着不填。以前这种事只能靠人自己一列列去检查。现在你可以直接把规则告诉 Claude Code，让它去扫飞书里的表。哪里空着，哪里不对，谁该补，它都能先帮你找出来。这个动作，就是在替你接一些原来得靠人盯着的事。

第三个场景，是处理数据。
以前飞书里的一大堆表，大多数时候就是存着。真要分析，还得自己导出来，再折腾一遍。现在它可以直接去读这些数据，再帮你整理成网页、日报，或者一个更顺手、更美观的页面。

所以我现在对这次更新的感觉很明确。开放 CLI 只是表层。真正变的，是软件开始给 AI 留入口了。未来甚至会为 AI 去设计新的功能。

很快你的工作方式就会变成，你开口，AI 去替你操作。人还是那个提需求的人，但真正去翻表、发消息、找数据、跑流程的，慢慢会换成 AI。

飞书这次真正让我觉得不一样的，就是这里。它不像是在单纯开放一个功能，它更像是在提前适应一件马上会发生的事，那就是软件不只是给人用，也是给 AI 用的。
