## 完整口播文案（定稿）

很多人以为 Claude Code 拼的是模型强弱。
但我这段时间用下来，真正把体验拉开差距的，可能只是 3 个很多人根本没用起来的隐藏命令。
其中有的命令，负责人发出来之后在 X 上讨论度非常高。
大家好我是 Alex，今天分享 3 个我自己使用频率非常高的 Claude Code 实用技巧。

第一个，`/btw`。
这个命令最适合什么场景？
就是 Claude 在执行一个很长的任务的时候。
比如抓取内容、排查 Bug、重构代码，可能一跑就是十几分钟。
这时候你脑子里其实会不断冒出一些小问题。
比如它现在到底在怎么抓？执行流程走到哪一步了？为什么这里要这么改？
但很多人又不敢问。
因为一问就怕把它带偏，或者把主任务的上下文弄脏。
`/btw` 最有价值的地方就在这。
它相当于给你开了一个侧边小窗口，让你可以临时插问这些问题，但不会把这些岔出去的问题混进主任务里。
所以它解决的不是 Claude 够不够聪明，
而是你终于可以在不中断主流程的情况下，满足自己的好奇心。

第二个，`/loop`。
这个命令我觉得特别被低估。
很多人现在用 Claude Code，还是一次性对话思维。
问一个问题，做一个任务，结束。
但 `loop` 不是。
它能让 Claude Code 变成一个持续替你工作的东西。
你可以让它每隔几分钟检查一次部署状态，也可以每天固定时间帮你巡检日志、盯某个信息源、或者跑一个重复性的检查任务。
说白了，它不是让 Claude 回你一次，
而是让 Claude 开始替你长期盯一类事情。
很多人今天和 Agent 的差距，不是不会提问，
而是根本没有把它放进自己的工作流里。
`/loop` 干的就是这件事。

第三个，`/insights`。
这个功能非常有意思。
它会根据你过去一段时间使用 Claude Code 的记录，生成一份 HTML 的可视化报告。
里面会看到什么？
你最常拿它做什么任务，哪些地方最容易翻车，你是不是经常把需求讲得不够清楚，甚至还会总结出你的一些使用习惯。
这个东西最有价值的地方在于，
它不是帮你完成任务，而是在帮你复盘你自己。
很多时候，不是 Claude 不够强，
是你的使用方式本身就有摩擦。
而 `/insights` 就像给你照了一面镜子，
让你第一次看清自己到底是怎么在用 Claude Code 的。

所以这 3 个命令如果你总结一下，它们分别解决的是 3 个特别真实的问题。
`/btw` 解决的是长任务里你想插问又不敢问。
`/loop` 解决的是很多事情你明明想交给 AI，但它没办法持续替你盯。
`/insights` 解决的是你用了很久 Claude Code，却从来没有认真复盘过自己的工作流。
这三个我都非常建议你去试一下。
尤其最后一个，很多你以为是模型的问题，最后看下来，其实是你自己的用法问题。
这 3 个命令我放评论区，大家直接收藏。
