# 小红书图文转化方法论

> 目标：当一条内容或素材已经深化完成后，快速判断它是否适合转成小红书图文；如果适合，应该匹配哪种模版、补哪些素材、用什么写法做成图文包。

---

## 核心原则

小红书图文不是“把视频稿切成几张图”。

它更像一篇被拆成多页的文章，核心是：
- 每一页只推进一个任务
- 主价值承载在页卡，不在正文区
- 标题负责点击，页卡负责收藏，正文负责补背景和承接评论

因此，图文转化要先判断：
1. 这条内容是不是适合“分页阅读”
2. 它应该属于哪种图文类型
3. 现有素材够不够支撑页卡
4. 还缺不缺外部观点、案例、截图或数据

---

## 第一步：判断能不能转图文

### 适合转图文的内容

符合以下任意 3 条以上，优先转图文：

1. **有明确带走感**
   - 读者看完能截图、收藏、照着做
   - 如：方法、清单、步骤、结论、判断框架、工具入口

2. **能拆成 5 个以上独立页卡**
   - 每一页能单独承担一个小任务
   - 如：定义、类比、案例、步骤、对比、风险、结论

3. **不依赖口播情绪和语气**
   - 去掉声音之后，内容依然成立
   - 如果内容价值严重依赖你的表情、语气、讲述节奏，优先保留视频

4. **适合搜索或收藏**
   - 用户会因为“以后要用”而收藏
   - 用户会因为“我想查这个”而点进来

5. **能补充静态证据**
   - 截图、对比表、案例列表、流程图、卡片总结

### 不适合直接转图文的内容

符合以下任意 2 条以上，先不要直接转：

1. **纯情绪判断**
   - 只有态度，没有步骤、框架、场景

2. **纯口播故事**
   - 靠讲述节奏和情绪感染成立，拆成页卡后只剩流水账

3. **没有中段承接**
   - 只有一个观点，没有例子、没有拆解、没有方法

4. **同一页要讲太多事**
   - 无法形成清晰页卡分工

5. **内容价值已经过时**
   - 热点窗口已过，补成图文也没有搜索价值

### 判定结论

- `强适合`：满足 4-5 条，直接进入图文生产
- `可转化`：满足 3 条，需要补素材再进入图文生产
- `弱适合`：只满足 1-2 条，不建议直接转图文

## 第一步补充：先过时效性筛选

不是所有适合图文的内容，都适合“现在”做图文。

在决定转图文前，必须先判断它属于哪种时效类型。

### 1. 常青型

**定义**
- 3 个月后仍然成立
- 核心价值不依赖单一发布时间

**适合你的题材**
- `API 是什么`
- `提示词方法`
- `反向提问工作流`
- `普通人怎么理解某个底层概念`

**处理方式**
- 优先转图文
- 可以反复迭代
- 重点打搜索和收藏

### 2. 半衰期型

**定义**
- 结论会保留，但细节可能 2-6 周内变化
- 适合做，但做之前要先补最新信息

**适合你的题材**
- `模型横评`
- `工具推荐`
- `价格 / 入口 / 套餐对比`
- `某个赛道最新打法`

**处理方式**
- 先做“信息刷新”
- 再转图文
- 过期信息不允许直接沿用

### 3. 热点事件型

**定义**
- 强依赖发布时间
- 热点窗口通常只有 `48 小时 - 14 天`

**适合你的题材**
- 新模型刚发布
- 某个工具刚爆
- 某次事件、某条新闻、某个官方更新

**处理方式**
- 超过窗口期后，不直接转“热点教程”
- 如果仍要做，必须改成：
  - `认知升级`
  - `机会判断`
  - `复盘总结`
- 不能再用“最全指南”“刚发布怎么用”这类强时效包装

### 时效判断规则

- `常青型`：直接进图文流程
- `半衰期型`：先刷新信息，再决定
- `热点事件型`：超过窗口后，不按原题转，必须改写角度

### 你的内容里常见的时效判断

- `API`：常青型
- `提示词技巧`：常青型
- `OpenClaw 具体技巧 / SOP`：半衰期型偏常青
- `模型横评`：半衰期型
- `Seedance 2.0 上手指南`：热点事件型，过窗口后不建议按原题再转

## 第二步：匹配图文模版

### 模版 A：认知升级型

**适合内容**
- 新概念科普
- 新范式判断
- 工具底层逻辑
- 抽象问题翻译给普通人

**适合你的题材**
- `API`
- `A2A`
- `OpenClaw 背后的底层逻辑`

**标题钩子**
- `X让我意识到，Y才是未来`
- `你以为大家在聊X，其实真正重要的是Y`
- `普通人迟早都得懂X`

**结构**
1. 强判断开头
2. 为什么这件事值得你看
3. 用类比翻译概念
4. 讲 2-4 个真实场景
5. 解释为什么这会影响普通人
6. 给出下一步动作

**字数建议**
- 页数：`10-12 页`
- 页卡：`180-260 字/页`
- 正文：`200-400 字`

**写法**
- 用类比，不堆术语
- 用场景，不空讲趋势
- 用“过去 vs 现在”的对比制造认知跃迁

### 模版 B：方法教程型

**适合内容**
- 一个 prompt
- 一个工作流
- 一个技巧
- 一个高频问题的解决方案

**适合你的题材**
- `提示词技巧`
- `反向提问工作流`
- `API Key 怎么拿`
- `某个工具怎么配`

**标题钩子**
- `这个X太厉害了：帮你Y`
- `被N万人围观的X到底有多强`
- `帮你写X的X`

**结构**
1. 痛点开头
2. 方法亮相
3. 为什么旧方法不好用
4. 新方法怎么做
5. 演示或实测
6. 你自己的优化版
7. 最终结论

**字数建议**
- 页数：`8-12 页`
- 页卡：`220-300 字/页`
- 正文：`150-350 字`

**写法**
- `before / after`
- 步骤拆解
- 给截图、代码块、结果图
- 讲“怎么做”，也讲“为什么这样做有效”

### 模版 C：清单资源型

**适合内容**
- 工具合集
- skills 合集
- 入口汇总
- 小白起步包

**适合你的题材**
- `最值得接 API 的几个工具`
- `普通人最该先懂的 AI 基础设施`
- `某赛道最值得留的工具清单`

**标题钩子**
- `2026用上这N个X，你就...`
- `我筛了很久，真正值得留的只有这N个`
- `别乱装了，先用这几类`

**结构**
1. 总论断
2. 分类 1
3. 分类 2
4. 分类 3
5. 每类补“适合谁”
6. 最后升维成一个使用原则

**字数建议**
- 页数：`10-14 页`
- 页卡：`250-350 字/页`
- 正文：`100-300 字`

**写法**
- 分类而不是堆砌
- 每项都给一句“为什么值得留”
- 每一类都说明适用人群

### 模版 D：身份赋能型

**适合内容**
- 给普通人、小白、非技术人建立进入感
- 打掉使用门槛
- 让某类人觉得“这和我有关”

**适合你的题材**
- `普通人为什么也该懂 API`
- `不会编程的人怎么开始用 AI`
- `学不会不是因为你笨，而是因为入口错了`

**标题钩子**
- `X类人才是Y最大的受益者`
- `不会X的人，反而更该学Y`
- `别再把这件事想复杂了`

**结构**
1. 身份判断
2. 3-5 个认知纠偏
3. 5-10 个起步 tips / ideas
4. 没灵感怎么办
5. 外部例子或截图做佐证

**字数建议**
- 页数：`6-8 页`
- 页卡：`250-350 字/页`
- 正文：`50-200 字`

**写法**
- 去恐惧
- 去术语
- 第二人称强
- 多用“你不是不会，是入口不对”

### 模版 E：机会判断型

**适合内容**
- 创业机会
- 商业机会
- 赛道判断
- 一个技术背后最值钱的那一层

**适合你的题材**
- `OpenClaw wrapper`
- `AI 执行力如何变成服务`
- `普通人怎样切入某个 AI 机会`

**标题钩子**
- `我看到一个用X赚钱的方法`
- `X最值钱的不是X，而是Y`
- `真正的机会藏在这层门槛里`

**结构**
1. 为什么我被这件事震到
2. 机会是什么
3. 为什么这门生意成立
4. 谁会买
5. 3-5 个具体方向
6. 为什么是现在

**字数建议**
- 页数：`10-14 页`
- 页卡：`250-320 字/页`
- 正文：`400-1200 字`

**写法**
- 一定写具体买家
- 一定写具体工作流
- 一定写具体交付
- 不能只写“前景很好”

### 模版 F：观点转译型

**适合内容**
- 播客、文章、演讲摘要
- 行业判断
- 热点后的理解

**适合你的题材**
- `某大佬对 AI 的判断`
- `某次大事件之后我的结论`
- `为什么某个观点你该重视`

**标题钩子**
- `某人最新X分钟播客：结论`
- `不瞒大家，我昨晚被 AI 刺激到了`
- `看完这件事，我反而更确定了`

**结构**
1. 情绪或权威入口
2. 2-3 个核心判断
3. 证据或案例
4. 把问题抛回读者

**字数建议**
- 页数：`4-6 页`
- 页卡：`180-260 字/页`
- 正文：`300-800 字`

**写法**
- 不是复述原话
- 是翻译成“这对普通人意味着什么”
- 最后要回到读者自己

---

## 第三步：盘点现有素材够不够

决定转图文后，先盘点手里有没有以下 5 类素材：

1. **你的核心观点**
   - 这条内容最重要的一句话是什么

2. **可拆页的结构**
   - 至少能拆出 5 个以上页卡任务

3. **场景或例子**
   - 没有场景，图文很难撑住中段

4. **静态证据**
   - 截图、对比表、列表、时间线、框架图

5. **结论或 CTA**
   - 看完之后，读者该怎么理解、怎么做、怎么看下一篇

若缺少第 3 或第 4 类，先补料，不要硬做图文。

## 第三步补充：先判断图片分布，而不是最后再找图

小红书图文不是“先写字，再临时配图”。

图片分布必须在写页卡之前就决定，因为不同模版需要的图片频率完全不一样。

### 对标样本里的图片分布规律

基于已拆解的 7 条对标笔记，可归纳出以下规律：

#### 1. 认知升级型
- **文字页占比**：`70%-80%`
- **图片/证据页占比**：`20%-30%`
- **图片频率**：平均每 `3-4 页` 出现一次
- **需要的图片类型**：
  - 权威媒体 / 官方原文截图
  - 产品界面截图
  - 简单关系图 / 类比图

#### 2. 方法教程型
- **文字页占比**：`50%-65%`
- **截图/演示页占比**：`35%-50%`
- **图片频率**：平均每 `2-3 页` 出现一次
- **需要的图片类型**：
  - prompt / 配置 / 代码块截图
  - 工具操作界面
  - before / after 对比图
  - 结果图

#### 3. 清单资源型
- **结构化文字页占比**：`70%-90%`
- **截图页占比**：`10%-30%`
- **图片频率**：低频，以分类切换处或结尾出现
- **需要的图片类型**：
  - 分类表格
  - 评分表
  - logo / 图标
  - 极少量产品截图做证明

#### 4. 身份赋能型
- **文字页占比**：`75%-90%`
- **图片页占比**：`10%-25%`
- **图片频率**：低频，主要用于增强代入感
- **需要的图片类型**：
  - 典型场景截图
  - 外部参考帖 / 推文截图
  - 一张“起步清单”或“自测卡”

#### 5. 机会判断型
- **文字页占比**：`75%-90%`
- **图片页占比**：`10%-25%`
- **图片频率**：低频，不依赖大量截图
- **需要的图片类型**：
  - 新闻 / 推文 / 原文来源截图
  - 工作流示意图
  - 场景 mockup
  - 极少量案例界面

#### 6. 观点转译型
- **文字页占比**：`50%-80%`
- **图片证据页占比**：`20%-50%`
- **图片频率**：高于认知升级型，通常每 `1-2 页` 要有一次视觉证据
- **需要的图片类型**：
  - 事件截图
  - 节目 / 产品 / 新闻现场图
  - 原话引用卡

### 先决定图片策略，再决定页卡密度

可以直接按下面这套方法判断：

1. 如果这条内容主要靠“解释”
   - 用 `低频图片`
   - 重点做文字页

2. 如果这条内容主要靠“教会”
   - 用 `高频截图`
   - 每 2-3 页必须有操作图或结果图

3. 如果这条内容主要靠“证明”
   - 用 `证据截图`
   - 不要只用抽象排版

4. 如果这条内容主要靠“收藏”
   - 用 `表格 / 清单 / 总结卡`
   - 比真实截图更重要

### 图片不够时怎么判断

如果一条内容缺图片，不代表不能做图文，要先判断缺的是哪一类：

- 缺 `证据图`：去补官方截图、媒体截图、评论截图
- 缺 `操作图`：去补工具界面、设置页、结果页
- 缺 `结构图`：自己做表格、流程图、关系图
- 缺 `场景图`：补典型使用场景或类比视觉

### 对你的直接要求

你之后做图文时，至少先回答这 3 个问题：

1. 这条图文总共几页
2. 其中多少页是纯文字页，多少页是截图页，多少页是表格 / 结构页
3. 我缺的图片是“证据图”“操作图”还是“结构图”

没回答这 3 个问题前，不进入成稿。

## 第四步：补网上观点，但不要“堆观点”

网上观点只用来做 3 件事：

1. **证明**
   - 证明你的判断不是空想
   - 例如：官方公告、权威媒体、产品截图

2. **对比**
   - 让你的观点更锋利
   - 例如：别人怎么理解，你为什么不一样

3. **放大**
   - 把你原本的素材从“个人经验”升级成“更大的趋势”

### 网上补料的优先顺序

1. 官方原始信息
2. 权威媒体或产品文档
3. 对标账号爆款图文
4. 评论区高频提问

### 网上补料的使用规则

- 不要一篇里塞 5 个外部观点，`1-2 个` 就够
- 外部观点不能抢走你的主体表达
- 外部材料优先服务“页卡中段”，不要都堆在开头

---

## 第五步：图文生产顺序

正确顺序是：

1. 判定是否适合转图文
2. 匹配模版
3. 列出已有素材
4. 找出缺口
5. 补网上观点 / 证据 / 截图
6. 先写页卡结构
7. 再写正文
8. 最后写标签

不要先写正文，再硬拆页。

---

## 第六步：标准输出格式

当用户说「把这条改成小红书图文」时，标准输出应包含：

### 1. 转化判断
- `强适合 / 可转化 / 弱适合`
- 原因 2-4 条

### 2. 模版匹配
- 使用哪个模版
- 为什么是这个模版，而不是别的

### 3. 素材盘点
- 已有素材
- 缺失素材
- 建议补什么

### 4. 网上补料建议
- 需要补哪类观点
- 需要补哪类证据
- 需要补哪类截图

### 5. 图文页卡骨架
- 第 1 页做什么
- 第 2 页做什么
- 第 3 页做什么
- ...

### 6. 正文 / 标签
- 正文不是逐页复述
- 标签优先服务搜索

## 第七步：对你账号的直接适用规则

### 你的内容里优先转图文的方向

1. `API`
   - 优先转 `认知升级型` / `方法教程型` / `清单资源型`

2. `提示词技巧`
   - 优先转 `方法教程型`

3. `OpenClaw`
   - 优先转 `机会判断型` 或 `具体场景教程型`
   - 不要优先转成纯观点吐槽

4. `学了AI为什么赚不到第一笔钱`
   - 不适合做泛口播转图文
   - 应改成 `身份赋能型 + 机会判断型`

### 你的图文常见误区

1. 想把视频稿直接切页
2. 一页里同时讲 3 件事
3. 正文重复页卡内容
4. 只有判断，没有中段承接
5. 只讲道理，不给读者带走感

---

## 最终判断标准

一条图文如果满足以下 4 条，才算真正可发：

1. 第 1 页有点击理由
2. 中段有收藏理由
3. 最后有评论或下一篇钩子
4. 整体看起来像“分页文章”，不是“视频字幕截图”
